Lab Assignment 2

Create your own spatial data and map it

Geographic information is the foundation of Web GIS. In this exercise, create your own simple dataset and display it in a web map.

For example, choose four restaurants you like in a city you’ve visited and record each restaurant’s location. In this case, your spatial dataset would be a combination of:

  • non-spatial data (restaurant name, description of the food and atmosphere) and
  • spatial data (latitude and longitude).

You may use the restaurants example or choose your own topic. This tutorial uses four landmarks in Baton Rouge as an example, with placeholder text and images from Wikipedia.

Add your data to a map in ArcGIS Online and display it in a Story Map.

Data

Create two “layers” of data: a point layer and a polygon layer.

Example of point and polygon features on a map.

Point layer

Your first dataset will be in the simple “comma separated values” (CSV) file format and will contain the names and descriptions of several places of interest along with their latitude and longitude coordinates (points). This exercise will create a CSV from a spreadsheet using Microsoft Excel, but any text editor will also work.

Our dataset will have attributes such as “name” and “description”. This exercise will use the Story Map Tour template in ArcGIS Online, so the attributes must match the template. Use the following attribute names:

  • name - a short, descriptive title for the location
  • description - a sentence or a few describing the location
  • icon_color - R, G, B, or P
  • lat - the location’s latitude in decimal degrees (e.g., 30.12345)
  • long - longitude in decimal degrees (e.g., -90.12345)
  • pic_url - The web address of an image you want to represent the location
  • thumb_url - The web address of a smaller preview image

To see a completed CSV for reference you can download this example CSV.

Polygon layer

In Part 3 we will create a polygon layer. It’s up to you what features you draw to complement your point layer. In the restaurant example, you could draw outlines around the buildings.

We will use geojson.io to draw the features on a map and then save them in GeoJSON format to import into ArcGIS Online.

Process

Part 1: Data prep

  1. Start Excel with a blank spreadsheet.

  2. Type an attribute name in each column on the first row, using the attribute names specified above.

    Example of a spreadsheet with attribute names in each column.

  3. Choose at least 4 places (4 restaurants, 4 national parks, 4 countries, etc.) to use for this exercise and research enough about them to fill in the attributes in the CSV. To find the latitude and longitude of your places, use the Measure tool in ArcGIS Online.

  4. After completing the spreadsheet, click File > Save as and for the file type select CSV (Comma delimited). You now have geographic information ready to be added to a map.

Part 2: Create the Story Map

  1. Sign into ArcGIS Online and open the Map Viewer with a new map.
  2. Click Add > Add Layer from File, browse for the CSV file on your computer, and click Import Layer. This will upload a copy of your file to ArcGIS Online.
  3. Click Cancel in the Change Style pane on the left side of the Map Viewer.

    The points should now be displayed on the map.
  4. Click the Save button and give your map a title, tags, and summary. This will save a new Web Map item in your Content.
  5. Click the Share button.
  6. Check the box for Everyone (public). This changes the map’s viewing permission from private to public.
  7. Click the Create a Web App button.
  8. From the selection of configurable apps, choose the Story Map Tour template and click Create Web App. You can use the same title, tags, and summary as the map, then click Done.
  9. You should now see your web app instead of the Map Viewer. This is the Configure Web App page, where you can change settings for things like colors and the layout.
  10. Go to your Content to see that you have new Web Map and Web Mapping Application items.

Part 3: Create another dataset in GeoJSON format

In this part you will draw your own features on a map and save the layer as a GeoJSON file.

  1. Go to geojson.io and pan and center the map on the place you chose for your dataset in Part 1.
  2. In the drawing tools, click the Draw a polygon or Draw a rectangle tool. Draw shapes on the map that are appropriate for your point layer, e.g., building footprints.
  3. After drawing your polygon(s), you can edit the attributes by either clicking the features on the map or clicking the Table tab on the top menu.
  4. Create a new column called “name” and on each row in the table, type/paste an appropriate name, depending on what features you chose to draw. For example, it could be the restaurant’s name.
  5. Go to Save > GeoJSON and the file will download to your computer.

Part 4: Add GeoJSON to your map.

  1. In ArcGIS Online, go to your Content and open your map (not app) from Part 2.
  2. In the Map Viewer, click Add > Add Layer from File, browse for the .geojson file on your computer, and click Import Layer.

  3. You should see the polygons on the map. When prompted to change the style, you can choose your own, such as removing the fill color so that only your polygon borders are visible. Click OK and Done to accept your style changes. You can see your symbols in the contents pane.

    Contents pane in ArcGIS Online, showing the symbology of a layer.

  4. Click on the More Options button under the polygon layer and choose Create Labels.

    The More Options icon (three blue dots) gives you access to more settings, including the option to turn on labels.

    This will display text on the map for each feature. The text comes from one of the layer’s attributes. In this example, choose the name attribute.

  5. Save the map. The changes will be automatically reflected in your Story Map from Part 2.

  6. Go to your Content and click on the item for your Story Map app to see its “Item Details” page:

    Example of an Item Details page in ArcGIS Online.

    The green progress bar on the right side of the page indicates how many of the metadata fields you have completed, e.g., Summary, Description, and Tags.

    From this page you can click View Application to launch the app, or click Configure App to make further changes. The app configuration only makes changes to the app, not the underlying map. The map would have to be edited in the Map Viewer again.
